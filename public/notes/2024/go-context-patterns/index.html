Go Context Cancellation Patterns

Common patterns for handling context cancellation in Go:

Pattern 1: Select with Context

func doWork(ctx context.Context) error {
    for {
        select {
        case <-ctx.Done():
            return ctx.Err()
        default:
            // do work
        }
    }
}

Pattern 2: Pass Context Down

Always pass context as the first parameter:

func processRequest(ctx context.Context, data Data) error {
    // Check context before expensive operations
    if err := ctx.Err(); err != nil {
        return err
    }
    
    return callDownstream(ctx, data)
}

Pattern 3: WithTimeout for API Calls

ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
defer cancel()

resp, err := httpClient.Do(req.WithContext(ctx))

Remember: Never store context in structs. Pass it explicitly to functions that need it.
